# Contributing

Contributions are welcome! 

## Development

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Open a pull request

## Guidelines

- Keep the demo simple and educational
- Prefer small, focused changes
- Include clear commit messages
- Avoid breaking the quick-start flow

## Security

If you discover a security issue, please open an issue with details.

Thank you for helping improve VFA-Lab!
